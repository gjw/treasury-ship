== Type Safety

* *522 → 363 violations* (30.4% reduction, target was 25%)
* 5 implementation passes: strict flags, cast elimination, `any` removal, `ApiError` class
* Replaced 100 unsafe `as` casts with type guards and discriminated union narrowing
* Eliminated 63 `any` types with proper interfaces (`SqlParam`, `TipTapNode`, `ApiIssueResponse`)
* Zero `@ts-ignore` or `@ts-nocheck` in the codebase — before or after
* Lesson: getting types right from day one is 10x easier than unwinding sloppy types after other code depends on them

== Runtime Error Handling

* *Gap 1 — Data Loss:* `persistDocument()` called without `.catch()` in collaboration server
** User edits silently lost if DB write fails on disconnect
** Fix: catch + single retry + structured logging
* *Gap 2 — Silent Crashes:* No `process.on('unhandledRejection')` handler
** Any unhandled async error killed the server with zero log output
** Fix: process-level handlers with error logging
* *Gap 3 — Stuck UI:* 3 of 4 sprint reconciliation mutations had no `onError`
** Button disabled permanently on API failure, no feedback to user
** Fix: `onError` resets pending state; global toast handler shows error

* Lighthouse scores: already 96-100 pre-fix (1 failed audit = only -4 points)

== Philosophy

* *Conservative:* No stack changes, no new dependencies, no behavioral changes
* *Safe:* Every fix verified with `pnpm type-check` and `pnpm test`
* *Measurable:* Before/after proof for every category — no claims without data
* *Honest:* Skipped findings documented with rationale, not hidden
* *Targeted:* Fix the root cause, not the symptom — depth over breadth
